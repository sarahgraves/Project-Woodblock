App Structure & Navigation

App Structure

Where possible, iHeartRadio apps should use our Global User Experience (GUX) structure. This structure ensures that users coming from other platforms can find their way around based on past experience. The GUX structure includes the following components:

Now Playing

This screen is where users will spend the vast majority of their time. It should contain all required controls for both Live and Custom audio streams and all required metadata. Additional details and content that can make the experience more engaging are encouraged. You can find out more about Now Playing requirements here.

Home Screen

The Home screen is the primary location for merchandised content that our programming team is responsable for populating. It is also responsible for communicating the primary value proposition of iHeartRadio. You can find out more about Home Screen requirements here.

Find

The Find section is primarily a browse-able catalog of available radio stations including all Live stations, featured Custom stations, and iHeartRadio Originals. Users should be able to find a station by location, genre, or type. You can find out more about Find section requirements here.

Create

The Create section is used to educate the user in what a Custom station is and how to create one. You can find out more about Create section requirements here.

Search

The Search functionality should allow users to quickly find any radio station, artist or song, using parameters ranging from zip code to name. You can find out more about Search functionality requirements here.

Profile

The user's Profile section will contain all information associated with a users account including, My Stations, Thumbed Up Songs, and friends. In many cases some of this information may be broken out and displayed in multiple parts of the app. You can find out more about the Profile section here.

Options

The Options section of the app will contain Account Settings, legal information, and about documents. In some cases this content can be broken into separate sections. You can find out more about the Options section here.

Navigation

Find or Create

iHeartRadio apps use primary navigation as a way to reenforce the app's value proposition. Namely the users ability to "Find over 1,000 Live radio stations" or "Create your own Custom radio station". Not all platform norms will support this pattern but it should be applied where possible.

Restrictive User Interfaces

On platforms where alphanumeric inputs are restricted on account of safety or form factor concerns, this navigation structure can be modified or reduced.
